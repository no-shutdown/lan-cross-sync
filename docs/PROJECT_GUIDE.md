# 项目说明

## 项目定位

LAN Cross Sync 是一个基于 Tauri v2 的桌面应用，用于在同一局域网内的 Windows 和 macOS 设备之间同步剪贴板内容和传输文件。

当前版本是可运行的 MVP，默认面向可信的家庭或办公局域网。项目目前没有实现互联网中继、NAT 穿透或端到端加密，不应将它描述为可以抵御局域网攻击的安全通信工具。

## 当前已实现

- 通过 UDP 广播发现同一局域网内的应用实例，默认发现端口为 `45731`。
- 使用 6 位临时配对码完成配对请求、响应和确认；配对记录会持久化。
- 只允许已配对设备进入业务连接，TCP 传输层支持协议版本检查、心跳、超时和断线重连。
- 同步文本和图片剪贴板，并通过内容哈希避免回写形成同步环路。
- 每个已配对设备可以单独关闭接收剪贴板；没有已配对且在线的目标时，不会读取、发送或回写本机剪贴板内容。
- 支持单文件、多文件和目录拖放传输，保留目录结构，并显示传输进度。
- 图片剪贴板单项上限为 4 MiB，文件传输清单最多包含 10,000 个条目。
- 文件接收使用安全相对路径、符号链接拒绝和清单条目数量限制，避免将文件写出到目标目录之外。
- 接收文件先写入目标目录下的 `.lan-cross-sync-staging` 临时目录，完成并校验后再移动到最终位置。
- 传输失败、取消、连接断开以及应用下次启动时，会清理索引中的临时传输目录，减少孤立的 `.part` 文件。
- 默认使用简体中文界面，也可以切换英文；界面语言会持久化。
- 支持系统托盘、关闭窗口隐藏到托盘、开机启动开关、原生文件选择和拖放。
- Windows 已支持 NSIS 安装包和 MSI 安装包；macOS 构建配置已包含 `.icns` 图标，但必须在 macOS 或 macOS CI 上生成 macOS 包。

## 主要目录

```text
src/                  React + TypeScript 前端和界面文案
src-tauri/src/        Rust 业务逻辑、网络、剪贴板和文件传输
src-tauri/tauri.conf.json
                      Tauri 窗口、开发服务器和打包配置
docs/PROJECT_GUIDE.md 当前项目说明
docs/BUILD_AND_TEST.md
                      安装、运行、验证和打包手册
docs/superpowers/     历史设计与实施记录，不作为当前状态清单
```

Rust 侧模块职责大致如下：

- `discovery.rs`：局域网发现和设备端点维护。
- `pairing.rs`：配对码状态和配对消息处理。
- `transport.rs`：TCP framing、授权握手、心跳和重连。
- `clipboard.rs`：文本/图片剪贴板监听、发送、接收和去环。
- `file_transfer.rs`：清单、分块、路径校验、进度和临时目录清理。
- `settings.rs`、`registry.rs`：本地设置和设备状态。
- `commands.rs`、`lib.rs`：Tauri 命令、事件和应用生命周期。

## 当前未实现或有明确限制的内容

- 配对后的密钥交换和端到端加密。
- 断点续传。中断后需要重新传输整个任务；新版本临时目录会被清理。
- 带宽限制、传输队列和更细粒度的冲突处理。
- 剪贴板历史、历史恢复和冲突选择界面。
- 通过原生文件选择器选择目录；目录拖放已经支持。
- 前端没有暴露发送中的主动取消按钮，底层取消和接收端拒绝流程已存在。
- 互联网同步、云端中继和 NAT 穿透。
- 发布用的 Windows 代码签名、macOS Developer ID 签名和 notarization。
- 当前工作区已完成 Windows 构建验证，但仍需要两台真实设备完成 macOS/Windows 双机验收。

## 数据和权限边界

- 本地设置保存于 Tauri 应用配置目录。Windows 通常为 `%APPDATA%\\com.local.lancrosssync\\settings.json`。
- 活跃接收传输的索引保存于 Tauri 应用缓存目录的 `active-transfers.json`。
- 文件临时目录位于用户选择的目标目录内，目录名为 `.lan-cross-sync-staging`。
- Windows 防火墙需要允许应用在专用网络中进行局域网通信；公共网络不建议放行。
- macOS 首次运行时可能需要允许本地网络、剪贴板或通知相关权限，具体提示取决于系统版本和签名状态。
- 配对码用于建立当前的信任关系，但不等同于密码学身份认证。请只在可信局域网内配对。
