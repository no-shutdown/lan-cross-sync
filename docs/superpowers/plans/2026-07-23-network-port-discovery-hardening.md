# 网络端口与发现加固实施计划

## 任务 1：先行测试

- 为 `PeerRegistry` 增加 UDP/TCP 端点分离测试。
- 为传输监听器增加首选 TCP 端口占用时回退的异步测试。
- 为网络状态结构增加序列化或状态构造测试。

## 任务 2：后端网络启动

- 增加固定 UDP 发现端口绑定入口。
- 增加 TCP 首选端口和系统分配端口的绑定逻辑。
- 在 Tauri 启动阶段完成两个端点预绑定；仅全部就绪时启动广播和后台网络任务。
- 将实际 TCP 端口保存到本地设置，并向前端提供网络状态。

## 任务 3：端点路由

- 将注册表的单一端点拆为发现端点和传输端点。
- 配对命令使用发现端点，TCP 连接和保活使用传输端点。
- 保留应用层 `Hello/HelloAck` 校验，拒绝不匹配协议或未配对设备。

## 任务 4：前端与文档

- 在仪表盘显示 UDP/TCP 端口、广播状态、端口回退和绑定失败状态。
- 增加简体中文和英文文案。
- 更新 README 和项目指南中的端口与启动说明。

## 任务 5：验证与交付

- 运行 `cargo fmt --check`、`cargo test`、`cargo check`、前端构建和 Tauri release 构建。
- 检查工作区差异，提交并推送到 `origin/main`。
